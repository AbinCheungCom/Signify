# Signify 宝塔面板一键部署教程（传统服务器）

> 企业家形象资产数字化系统 · Laravel 11 + Vue 3 + Inertia.js
> **兼容环境**：Nginx 1.28+ · MySQL 5.7+ · PHP 8.0+

---

## 环境要求（传统服务器）

| 项目 | 版本要求 | 说明 |
|------|----------|------|
| **PHP** | 8.0+ | 推荐 8.1，性能更好 |
| **MySQL** | 5.7+ | 需 InnoDB 引擎 |
| **Nginx** | 1.28+ | 或 Apache 2.4+ |
| **phpMyAdmin** | 5.0+ | 数据库管理（可选） |

**PHP 扩展（必须）：** `pdo_mysql`, `mbstring`, `openssl`, `fileinfo`, `curl`, `gd`

> ⚠️ **MySQL 5.7 兼容性说明**：Laravel 11 默认使用 enum 类型字段，但 MySQL 5.7 不支持。项目中已将 `status` 字段改为 `string` 类型替代，完全兼容 MySQL 5.7。

---

## 第一步：创建网站和数据库

1. 登录宝塔面板 → **网站** → **添加站点**
2. 配置：
   - 域名：填写你的域名或IP
   - 根目录：`/www/wwwroot/signify`
   - PHP版本：**选择 8.0** 或 **8.1**
   - 创建数据库：**勾选**，MySQL 版本选 **5.7**，设置数据库名/用户/密码

---

## 第二步：上传代码

### 方式 A：Git 克隆（推荐）

1. 进入网站根目录：
   ```bash
   cd /www/wwwroot/signify
   ```

2. 克隆仓库：
   ```bash
   git clone https://github.com/AbinCheungCom/Signify.git .
   ```

3. 如果提示目录非空：
   ```bash
   cd /www/wwwroot/signify
   # 先清空目录
   rm -rf ./*
   # 再克隆
   git clone https://github.com/AbinCheungCom/Signify.git .
   ```

### 方式 B：直接上传

1. 下载仓库 ZIP 包
2. 上传至 `/www/wwwroot/signify/` 并解压
3. 确保所有文件在网站根目录下（不要多层嵌套）

---

## 第三步：配置 .env

1. 复制环境配置模板：
   ```bash
   cd /www/wwwroot/signify
   cp .env.example .env
   ```

2. 编辑 `.env` 文件（可通过宝塔文件管理器或 SSH）：
   ```env
   APP_NAME="Signify"
   APP_ENV=production
   APP_DEBUG=false
   APP_URL=https://你的域名.com

   DB_CONNECTION=mysql
   DB_HOST=127.0.0.1
   DB_PORT=3306
   DB_DATABASE=signify          # 填你创建的数据库名
   DB_USERNAME=signify          # 填你创建的用户名
   DB_PASSWORD=你的数据库密码    # ⚠️ 务必使用强密码（12位以上，含大小写+数字+符号）

   SESSION_DRIVER=file
   FILESYSTEM_DISK=public
   ```

---

## 第四步：安装依赖

### 通过 SSH 终端执行

1. 进入网站目录：
   ```bash
   cd /www/wwwroot/signify
   ```

2. 安装 Composer 依赖：
   ```bash
   # 生产环境（不带开发依赖）
   composer install --no-dev --optimize-autoloader
   
   # 开发/测试环境（包含 phpunit 等开发依赖）
   # composer install --optimize-autoloader
   ```

3. 生成应用密钥：
   ```bash
   php artisan key:generate
   ```

4. 创建存储链接：
   ```bash
   php artisan storage:link
   ```

5. 运行数据库迁移（兼容 MySQL 5.7）：
   ```bash
   php artisan migrate --force
   ```

6. 设置权限：
   ```bash
   chown -R www:www /www/wwwroot/signify
   chmod -R 755 /www/wwwroot/signify/storage
   chmod -R 755 /www/wwwroot/signify/bootstrap/cache
   # 关键：限制 .env 文件权限，防止数据库密码泄露
   chmod 600 /www/wwwroot/signify/.env
   ```

7. 生产环境优化：
   ```bash
   php artisan optimize
   ```

8. 清除缓存：
   ```bash
   php artisan config:clear
   ```

## 第五步：配置网站

### 修改网站配置文件

1. 宝塔面板 → **网站** → 找到你的站点 → **设置** → **配置文件**

2. 删除默认配置，替换为：

```nginx
server {
    listen 80;
    server_name 你的域名或IP;
    root /www/wwwroot/signify/public;
    index index.php;

    charset utf-8;

    # 安全响应头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;

    # 请求日志
    access_log /www/wwwroot/signify/storage/logs/access.log;
    error_log /www/wwwroot/signify/storage/logs/error.log;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location = /favicon.ico {
        access_log off;
        log_not_found off;
    }

    # PHP-FPM 配置（传统服务器常见路径）
    # 方式1: Unix Socket（大多数宝塔面板）
    location ~ \.php$ {
        fastcgi_pass unix:/tmp/php-cgi.sock;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        include fastcgi_params;
        fastcgi_read_timeout 300;
    }

    # 方式2: TCP 连接（某些面板使用）
    # location ~ \.php$ {
    #     fastcgi_pass 127.0.0.1:9000;
    #     fastcgi_index index.php;
    #     fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    #     include fastcgi_params;
    # }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 30d;
        add_header Cache-Control "public";
    }

    # 禁止访问敏感文件
    location ~ /\.env {
        deny all;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }

    # Gzip 压缩（低配服务器推荐开启）
    gzip on;
    gzip_vary on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
}
```

> ⚠️ 注意：`fastcgi_pass` 的路径需要与服务器实际配置匹配。常见选项：
> - Unix Socket: `/tmp/php-cgi.sock` 或 `/tmp/php-cgi-82.sock`
> - TCP: `127.0.0.1:9000`
> 可在宝塔面板 PHP 设置中查看实际的连接方式。

3. 保存并重启 Nginx：
   - 宝塔面板 → **软件商店** → **Nginx** → **重启**

## 第六步：SSL 配置（可选但强烈推荐）

1. 宝塔面板 → **网站** → 你的站点 → **SSL**
2. 选择 **Let's Encrypt** 或上传已有证书
3. 开启 HTTPS 并强制跳转

## 常见问题

### 500 错误

1. 检查 `.env` 配置是否正确
2. 检查 PHP 错误日志：宝塔面板 → **日志** → **PHP错误日志**
3. 清除缓存：
   ```bash
   php artisan config:clear
   php artisan cache:clear
   ```

### 权限问题

```bash
chown -R www:www /www/wwwroot/signify
chmod -R 755 /www/wwwroot/signify/storage
chmod -R 755 /www/wwwroot/signify/bootstrap/cache
```

### 数据库连接失败

1. 确认数据库名/用户名/密码与宝塔创建的一致
2. 确认数据库用户有权限访问该数据库
3. 检查 `.env` 中 `DB_HOST` 是否为 `127.0.0.1`

### 静态资源 404

确认 `php artisan storage:link` 已执行，且 `/www/wwwroot/signify/public/storage` 链接指向 `storage/app/public`。

### MySQL 5.7 迁移失败

如果遇到字符集问题，检查 MySQL 配置：
```bash
# 登录 MySQL
mysql -u root -p

# 检查字符集
SHOW VARIABLES LIKE 'character%';

# 如需要，修改 /etc/my.cnf
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci
```

部署完成后的目录结构应如下：

```
/www/wwwroot/signify/
├── app/                    # 主程序
├── bootstrap/
├── config/
├── database/
├── public/
│   └── index.php          # 入口文件
├── resources/
├── routes/
├── storage/               # 需设置 755 权限
├── .env                   # 配置文件
├── composer.json
├── package.json
└── ...
```
